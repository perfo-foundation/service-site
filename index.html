Free performance review tool

Easy to use open source decision

We are creating open source project which is directed to provide you with a easy way to operate with the service and obtain all information in proper representation. We will have some comercial decisions in the future (some integrations and working through our cloud), but the whole service will be free all the time.

Secure way to operate with you data

Security is the most important thing nowadays. That's the reason why we are implementing service which carries about all security things. First of all, you are able to launch this application on your own service and control everything. Hence this is open source you can easily check that we don't have any malware or potentially breaches in the code.

Support is essential

We are ready to provide you with any support you only need to perform you daily basis operations with our software. Feel free to ask us for any help.

A few words about PERFO

Our goal is providing a great tool which allows everyone to apply any kind of performance review in their company or team. We know that a lot of companies just create their own decisions. But sometimes you might not have enough resources to create your own implementation. Or you just don't want to spend developers hours to create something like this. In this moments we are here to help you with PERFO.

We hope that every user of our service will find it as a long term tool. Becasue for develpers like us it's a honor when their product is widely used. We don't want to charge for our product. That's a reason why we distribute the product with no charge. In the future we are planning to create cloud platform which will be paid, but it still will be open source product which you will be able ti use wherever you want.

Our roadmap

We have a lot of thing to do. And here we will clarify some features that are related to the final consumers. Internal stuff is counting inside our project backlogs. Here you can find actual future features.

In the version 0.1 we are planing to implement basic services:

  • create teams
  • create users and employees
  • create departments
  • set users info (like skills, bio, etc, grades)
  • set users goals
  • basic permissions according to user role (manager, user, etc)
  • basic authorisation
  • mechanism of data exporting
  • basic statistics (users, teams, selections by grades, etc)
  • first UI version
